# Spam/Ham Text Classifier

A full-stack ML application that classifies text (emails, SMS, messages) as **spam** or **ham** (legitimate). Built with:

- **Model**: TF-IDF + Logistic Regression (scikit-learn) — 98.45% accuracy, 0.9834 F1 on held-out test data
- **Backend**: FastAPI REST API
- **Frontend**: React (Vite) + Tailwind CSS — interactive UI with live classification, confidence gauge, scan history, and model benchmarks
- **Deployment**: Docker Compose (2 containers: backend + frontend)

## Project structure

```
spam_classifier/
├── backend/
│   ├── dataset.csv          # training data (label, text)
│   ├── train.py             # trains the model, saves model.joblib/vectorizer.joblib/metrics.json
│   ├── main.py               # FastAPI app
│   ├── requirements.txt
│   └── Dockerfile            # trains the model at build time, then serves the API
├── frontend/
│   ├── src/
│   │   ├── App.jsx           # main UI
│   │   └── index.css         # design tokens + Tailwind
│   ├── nginx.conf            # serves the SPA + proxies /api to backend
│   └── Dockerfile             # multi-stage build (node -> nginx)
└── docker-compose.yml
```

## Run with Docker (recommended)

Prerequisite: Docker Desktop installed and running (on Windows, enable WSL 2 backend).

```bash
cd spam_classifier
docker-compose up --build
```

This will:
1. Build the **backend** image — installs dependencies, **trains the model from `dataset.csv`** (so the image is self-contained and reproducible), then starts the FastAPI server on port `8000`.
2. Build the **frontend** image — builds the React app, then serves it via Nginx on port `80`, proxying `/api/*` requests to the backend container.

Once both containers are up, open:

**http://localhost**

To stop:
```bash
docker-compose down
```

To rebuild after code changes:
```bash
docker-compose up --build
```

## API endpoints (backend, port 8000)

| Method | Endpoint              | Description                          |
|--------|------------------------|---------------------------------------|
| GET    | `/api/health`           | Health check                          |
| GET    | `/api/metrics`          | Training metrics (accuracy, F1, etc.) |
| POST   | `/api/predict`          | Classify a single text                |
| POST   | `/api/predict/batch`    | Classify up to 200 texts at once      |

Example:
```bash
curl -X POST http://localhost:8000/api/predict \
  -H "Content-Type: application/json" \
  -d '{"text": "Congratulations! You won a free prize, click here now!"}'
```

Response:
```json
{
  "label": "spam",
  "spam_probability": 0.93,
  "ham_probability": 0.07,
  "confidence": 0.93,
  "inference_time_ms": 1.2
}
```

Interactive API docs (Swagger UI) are auto-generated by FastAPI at:
**http://localhost:8000/docs**

## Running without Docker (local dev)

**Backend:**
```bash
cd backend
pip install -r requirements.txt
python train.py        # trains the model, saves model.joblib + vectorizer.joblib
uvicorn main:app --reload --port 8000
```

**Frontend:**
```bash
cd frontend
npm install
npm run dev
```
The Vite dev server proxies `/api` to `http://localhost:8000` (see `vite.config.js`).

## Notes on the model

- Dataset: ~10,000 labeled emails (balanced: ~53% ham, ~47% spam).
- Features: TF-IDF (unigrams + bigrams, 20,000 max features, English stop words removed).
- Two models were trained and compared — Multinomial Naive Bayes and Logistic Regression — and the better one (by F1 score) is automatically saved by `train.py`.
- Because the training data is **email-style** text (longer, conversational/forum-style), the model performs best on similar inputs. Very short SMS-style inputs may get lower-confidence predictions — this is a domain characteristic, not a bug. If you want stronger SMS performance, retrain with an SMS-style dataset (e.g. the UCI SMS Spam Collection) mixed in.
- Retrain anytime by replacing `backend/dataset.csv` (must keep `label`,`text` columns with `spam`/`ham` values) and re-running `python train.py` or `docker-compose build backend`.
